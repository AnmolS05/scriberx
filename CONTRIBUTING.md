# Contributing to ScribeRx

First off, thank you for considering contributing to ScribeRx! It's people like you that make ScribeRx such a great tool.

## Where do I go from here?

If you've noticed a bug or have a feature request, please check the [issues tab](https://github.com/AnmolS05/scriberx/issues) to see if there's already an open issue. If not, feel free to open a new one.

## Setting up your environment

1. Fork the repository
2. Clone your fork locally: `git clone https://github.com/your-username/scriberx.git`
3. Install dependencies: `npm install`
4. Setup your local environment file: `cp .env.example .env.local`
5. Add your Gemini API key to `.env.local`
6. Start the development server: `npm run dev`

## Making Changes

- Create a new branch for your feature/fix: `git checkout -b feature/your-feature`
- Make your changes
- Test your changes thoroughly
- Update the documentation (README.md, etc.) if necessary
- Ensure your code follows the existing style

## Committing

We use a conventional commit style. Please make sure your commits are clear and descriptive.

Examples:
- `feat: add new reporting feature`
- `fix: resolve issue with inventory calculation`
- `docs: update setup instructions`

## Submitting a Pull Request

1. Push your changes to your fork: `git push origin feature/your-feature`
2. Open a Pull Request against the `main` branch of the original repository
3. Describe your changes in detail in the PR description
4. Wait for review! We will get back to you as soon as possible.

Thank you for contributing!
